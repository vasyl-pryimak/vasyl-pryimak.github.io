> *"Реверб звучав добре. Але всі ноти — в тій самій тональності. Захотілось трохи розстроїти хвіст."*

Bitcrusher на A3 звучав цікаво — але як lo-fi ефект, не як частина реверба. A2 (decay) і A3 (bitcrusher) — по суті незалежні ефекти, жоден не відчувається як органічна частина реверба.

Захотілось щось інше. Щось що звучить як *частина* реверба, а не поверх нього.

У vintage spring reverb пружина фізично детюнить сигнал — через свою механіку хвіст звучить трохи інакше по висоті ніж прямий звук. Це і є ціль: **pitch shifter на хвіст реверба**.

**Ціль:** додати до реверба pitch shifter на A3. Не транспозицію, а детюнінг — ±5% від оригінальної частоти. Ефект як у spring reverb у старих апаратах: хвіст ледь відрізняється по висоті від прямого звуку. Живо.

## Теорія: як перемістити pitch у цифровому домені

Аналоговий спосіб — змінити швидкість плівки. Швидше → вище. Повільніше → нижче. Але плівка закінчується.

Цифровий спосіб — читати з кругового буфера з іншою швидкістю ніж записуємо.

```
запис:    1 семпл/такт   ──────────────────────────────→ (час)
читання:  0.95 семплів/такт ─────────────────────────→

різниця = голівка читання відстає від голівки запису
```

Якщо голівка читання відстає → вона "розтягує" сигнал у часі → ті самі частоти звучать довше → pitch вниз.

Якщо обганяє → "стискає" сигнал → pitch вгору.

### Що таке "голівка"

Візьмемо аналогію з касетним програвачем. У ньому є фізична голівка — маленький магніт який читає плівку в певній точці. Тут — те саме, але в цифровому вигляді: **голівка — це просто число**, індекс позиції в буфері звідки зараз читаємо семпл.

**Проблема з однією голівкою:** буфер кінцевий. Коли голівка читання доходить до кінця — вона стрибає на початок:

```
буфер:  [0][1][2]...[2046][2047][0][1][2]...
                               ↑
                          голівка читає тут

наступний такт: переходить на [0] — стрибок адреси
```

Стрибок адреси = різкий стрибок значення семпла = клацання в звуку.

**Рішення — дві голівки**, зсунуті на пів-буфера. Уяви дві стрілки на циферблаті, зсунуті на 180°:

```
голівка 1:  [----●-----------]  ← посередині, вага = 1.0
голівка 2:  [●---------------]  ← біля краю,  вага = 0.0

трохи пізніше:

голівка 1:  [----------●-----]  ← наближається до краю, вага падає
голівка 2:  [------●---------]  ← відійшла від краю,   вага зростає

коли голівка 1 стрибає:

голівка 1:  [●---------------]  ← тільки що стрибнула, вага = 0.0 (не чутно!)
голівка 2:  [---------●------]  ← посередині,          вага = 1.0
```

Голівка стрибає тільки коли її вага = 0 — тобто її в цей момент взагалі не чутно. Тому клацань немає. Це називається **Overlap-Add pitch shifting**.

## Circular buffer і phase accumulator

Буфер — 2048 семплів (блочна RAM). Голівка запису рухається рівно 1 семпл/такт:

```verilog
always @(posedge clk) begin
    if (we) begin
        mem[wr_ptr] <= audio_in;
        wr_ptr <= wr_ptr + 1'd1;
    end
end
```

Голівка читання — **phase accumulator**. 24 біти:
- `[23:12]` — ціла частина (номер семпла в буфері)
- `[11:0]` — дробова частина (для інтерполяції між семплами)

```verilog
reg [23:0] pha = 0;

always @(posedge clk) begin
    if (we) begin
        // крок = 1.0 - ratio
        // 1.0 в Q2.14 = 0x4000
        pha <= pha + $signed({ {10{16'h4000 < ratio}}, (16'h4000 - ratio), 2'b00 });
    end
end
```

Якщо `ratio = 0x4000` (рівно 1.0) → крок = 0 → `pha` не рухається відносно запису → немає pitch shift.

Якщо `ratio = 0x3C00` (0.9375) → крок позитивний → `pha` зростає → голівка читання відстає → pitch вниз.

Якщо `ratio = 0x4400` (1.0625) → крок від'ємний → `pha` зменшується → голівка читання обганяє → pitch вгору.

### Q2.14: fixed-point у FPGA

У FPGA немає "float". Числа 0..1 представляємо як integers з фіксованою позицією крапки.

**Q2.14** = 2 біти цілої частини + 14 біт дробової. Діапазон: 0.0000..3.9999.

```
0x0000 = 0.00000
0x2000 = 0.50000
0x4000 = 1.00000   ← точно 1.0
0x7FFF = 1.99994
```

Значення 1.0 = `1 << 14 = 0x4000`. Тому `0x4000 - ratio` дає нам `1.0 - ratio` — крок фази.

## Дві голівки читання

```verilog
wire [23:0] pha1 = pha;
wire [23:0] pha2 = pha + { 12'd2048, 12'd0 };  // +2048 семплів

// Адреси = позиція запису + зсув з фази
wire [ADDR_BITS-1:0] a1_0 = wr_ptr + pha1[23:12];
wire [ADDR_BITS-1:0] a2_0 = wr_ptr + pha2[23:12];
```

`pha2` завжди рівно 2048 "за" `pha1`. При `ADDR_BITS=11` (буфер 2048) — рівно пів-буфера.

## Лінійна інтерполяція (LERP)

Проблема цілочисельних адрес: коли `ratio` не дорівнює 1.0, голівка читання має рухатись по "дробових" індексах (наприклад, `10.45`). Якщо просто відкинути дробову частину і читати індекс `10` — голівка буде "застрягати" на одному семплі, а потім різко "стрибати" вперед. Це створює "сходинки" → aliasing → "рипіння".

**Приклад:** голівка стоїть на позиції `10.45`. В буфері:

```
індекс:   10      11
значення: 120     200
                ↑
           хочемо тут (10.45)
```

Без LERP — беремо просто `120` (ігноруємо `.45`).

З LERP — рахуємо точку між двома сусідніми:

```
результат = 120 + 0.45 × (200 - 120)
          = 120 + 0.45 × 80
          = 120 + 36
          = 156
```

На звуковій хвилі це виглядає так:

```
без LERP:               з LERP:

200 │      ┌──┐         200 │         ╱
    │      │  │             │       ╱
156 │      │  │         156 │     ╱  ←  10.45
    │      │  │             │   ╱
120 │──────┘  └──        120 │─╱────────
    └─────────────           └──────────
      10   11   12             10  11  12
```

Без LERP — сходинки, чути "рипіння". З LERP — плавна крива, чистий звук.

```verilog
function signed [15:0] lerp;
    input signed [15:0] v0, v1;
    input [11:0]        f;         // 0..4095 — дробова частина pha
    reg signed [27:0]   diff;
    begin
        diff = ($signed(v1) - $signed(v0)) * $signed({1'b0, f});
        lerp = v0 + diff[27:12];  // ÷4096 (нормалізація)
    end
endfunction

wire signed [15:0] tap1 = lerp(s1_0, s1_1, pha1[11:0]);
wire signed [15:0] tap2 = lerp(s2_0, s2_1, pha2[11:0]);
```

До цього: голівка читання рипіла. Після: гладенько.

## Кросфейд (windowing)

Дві голівки потрібні щоб стрибок не чувся. Але як саме вони "передають одна одній"?

**Кросфейд** — плавне перемикання гучності між голівками. В будь-який момент сума їх гучностей = 100%:

```
w1:  ╱╲    ╱╲    ╱╲
w2:   ╲╱    ╲╱    ╲╱
    ──────────────────
sum:  завжди 1.0
```

Голівка 1 йде від повної гучності → тиша → стрибає → знову зростає. Голівка 2 — зсунута на пів-буфера, робить те саме зі зміщенням. Голівка стрибає тільки коли її гучність = 0 — тому клацань не чутно.

**Windowing** — це конкретна форма кривої. `pha1[23]` (старший біт фази) визначає підйом чи спуск:

```verilog
reg [11:0] w1;
always @(*) begin
    if (pha1[23])
        w1 = ~pha1[22:12];  // спадає: 4095 → 0
    else
        w1 = pha1[22:12];   // зростає: 0 → 4095
end
wire [11:0] w2 = 12'hFFF - w1;  // сума завжди = 0xFFF
```

В будь-якій точці `w1 + w2 = 0xFFF ≈ 1.0`. Гучність не скаче — навіть у момент стрибка голівки.

```verilog
wire signed [31:0] mixed = ($signed(tap1) * $signed({1'b0, w1})) +
                           ($signed(tap2) * $signed({1'b0, w2}));

always @(posedge clk) begin
    if (we) audio_out <= mixed[27:12];
end
```

## Інтеграція з reverb: "pitched tail"

Pitch shifter можна поставити по-різному — і це кардинально змінює результат.

**Варіант 1: pitch shifter на весь сигнал**
```
audio_in → pitch_shifter → reverb → output
```
Pitch shift чуєш і на прямому звуці. Граєш ноту — вона вже звучить фальшиво. Не те.

**Варіант 2: тільки хвіст детюниться**
```
audio_in ──────────────────────────→ reverb (dry path)
         → pitch_shifter → reverb (wet path)
```
Прямий звук — без змін. Pitch shifter — тільки на мокрий сигнал. Чуєш чисту ноту + хвіст реверба який ледь "розплавлений" по висоті.

Саме так працює vintage spring reverb — пружина фізично детюнить через свою механіку. Ми імітуємо цю фізику. Ключова деталь: **dry path минає pitch shifter**.

```
audio_in → audio_dry (dry path, прямо в reverb_core)
         → pitch_shifter → audio_in (wet path)
                              ↓
                         reverb_core
                         (змішує dry + pitched wet)
```

Результат: чуєш чистий оригінал + ревербований хвіст з трохи зміненою висотою. Як у vintage spring reverb — пружина трохи детюнить через фізику.

## Що пішло не так

### 1. Фаза в неправильному форматі

Перша версія: крок фази обраховувався неправильно — не враховувалась різниця між Q2.14 (ratio) і внутрішнім форматом фази (12 біт дробу). Результат: голівка читання рухалась з неправильною швидкістю, pitch shift був непередбачуваний.

Фікс: явний зсув `(16'h4000 - ratio) << 2`.

### 2. Знак кроку фази

Коли `ratio > 0x4000` (pitch up), крок стає від'ємним. Але `pha` — unsigned. Фікс: явне знакове розширення:

```verilog
$signed({ {10{16'h4000 < ratio}}, (16'h4000 - ratio), 2'b00 })
```

### 3. Чотири читання за один такт

Для LERP потрібно читати два сусідніх семпли — і це для кожної з двох голівок. Тобто за один такт потрібно 4 читання з пам'яті:

```
голівка 1: читай [N],      читай [N+1]
голівка 2: читай [N+2048], читай [N+2049]
```

Block RAM — як бібліотека з одним вікном видачі: за раз можна попросити тільки одну книгу. Чотири запити одночасно — фізично неможливо.

Фікс: замість одного блоку RAM — чотири окремих, кожен відповідає за свій запит:

```
RAM_A → голівка 1, семпл [N]
RAM_B → голівка 1, семпл [N+1]
RAM_C → голівка 2, семпл [N]
RAM_D → голівка 2, семпл [N+1]
```

Чотири вікна — чотири читання одночасно. FPGA має достатньо блочної RAM щоб дозволити собі це.

## Результат

```bash
./reverb-sim input.wav output.wav --pitch=0.45 --wet=0.7 --decay=0.6
```

`pitch=0.5` → neutral.  
`pitch=0.4` → слабкий детюн вниз (~1 semitone).  
`pitch=0.3` → помітно нижче, хвіст звучить як "старий магнітофон".

З `--wet=0.8 --decay=0.7 --pitch=0.45` — ефект vintage spring reverb із ледь чутним "розплавленим" хвостом.

## Підсумок

| Компонент | Що робить |
|-----------|-----------|
| `pitch_shifter.v` | Overlap-add: 2 голівки читання, lerp, crossfade |
| Phase accumulator | 24-bit: [23:12]=адреса, [11:0]=lerp frac |
| Linear interpolation | lerp(s0,s1,frac) → усуває aliasing |
| Crossfade window | w1=трикутна хвиля по pha1[23:12], w1+w2=const |
| Q2.14 ratio | 0x4000=1.0, діапазон ~0x3C00..0x4400 (±5%) |

Block RAM: 1 × DEPTH=2048 × 16-bit = **1 M9K** (+ 3 M9K для reverb = 4 M9K з 30).
