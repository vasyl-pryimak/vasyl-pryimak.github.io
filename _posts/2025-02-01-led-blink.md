Після двох з половиною тижнів боротьби з інфраструктурою — нарешті можна писати код.

```verilog
module led_blink (
    input  wire clk,   // 50 MHz тактовий сигнал
    output reg  led    // вихід на світлодіод
);

reg [24:0] counter = 0;  // 25-бітний лічильник

always @(posedge clk) begin
    counter <= counter + 1;
    if (counter == 25_000_000) begin  // 0.5 секунди при 50 MHz
        counter <= 0;
        led <= ~led;  // перемикаємо LED
    end
end

endmodule
```

---

## Mindset shift для Java-розробника

```java
// Java: послідовні інструкції, процесор виконує одну за одною
counter++;
if (counter == 25_000_000) { led = !led; }
```

```verilog
// Verilog: опис схеми, все відбувається ПАРАЛЕЛЬНО
always @(posedge clk) begin
    counter <= counter + 1;    // ← ці дві операції відбуваються
    if (...) led <= ~led;      // ← одночасно в кожен такт
end
```

`<=` — це *неблокуюче присвоювання*. Всі значення обчислюються одночасно в поточному такті, а записуються — в наступному. Це не "присвоїти значення", це "поставити в черзі для наступного такту".

Якщо написати `=` (блокуюче) де треба `<=` — схема буде неправильна, але компілятор не попередить. 🙃
