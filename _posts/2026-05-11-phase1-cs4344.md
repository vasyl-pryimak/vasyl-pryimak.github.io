Ок, LED блимає. Наступна ціль — **440 Hz синусоїда** в навушники через CS4344 DAC.

**I²S за 30 секунд:** 4 дроти (MCLK, LRCLK, SCLK, SDIN), 24 біти на семпл, стерео, MSB першим. Промисловий стандарт цифрового аудіо.

---

Перший запуск — тиша.

Перевірив тактування логічним аналізатором — є. Перевірив SDIN — є сигнал. Перевірив код — все правильно. Тиша.

**Причина:** CS4344 — SMD чіп. Маленька мікросхема з ніжками, яка просто лежала на пінах breadboard без пайки. Контакт — суто механічний і ненадійний.

Взяв паяльник. 5 хвилин. Звук з'явився.

> 🔧 SMD на breadboard без пайки — це не підключення, це сподівання.

## Проблема 2: `$readmemh` в Quartus — пастка

Для синусоїди потрібна Sine LUT — 256 значень. Спочатку зробив .hex файл:

```verilog
reg [7:0] sine_table [0:255];
initial $readmemh("sine.hex", sine_table);  // ← в симуляції працює!
```

В Icarus Verilog симуляції — все ок. В Quartus (реальне залізо) — LUT ініціалізується нулями.

**Причина:** `$readmemh` — конструкція для симуляції, синтезатор її ігнорує.

Рішення — хардкодом:

```verilog
function [7:0] sine_lut;
    input [7:0] addr;
    case (addr)
        8'd0:   sine_lut = 8'd128;
        8'd1:   sine_lut = 8'd131;
        // ... 256 рядків
    endcase
endfunction
```

Так, 256 рядків. Зате працює завжди і скрізь.

## Проблема 3: Philips I²S ≠ Left-Justified

I²S має кілька варіантів, і різниця між ними — буквально **один такт**.

**Philips I²S** — розробив Philips у 80-х для своїх CD-плеєрів. Став найпоширенішим стандартом. MSB з'являється через **1 SCLK після** зміни LRCLK.

**Left-Justified** — паралельний стандарт тих самих часів. MSB йде одразу після зміни LRCLK, без gap. Зустрічається в деяких старших чіпах, DSP процесорах (TI, Analog Devices).

```
Left-Justified (мій перший варіант — неправильний для CS4344):

LRCLK  ─────────┐                   ┌────────
                 └───────────────────┘
SCLK   ──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──
          └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘
SDIN   ──[MSB][B6 ][B5 ][B4 ][B3 ][B2 ][ B1]
         ↑
         MSB одразу після зміни LRCLK


Philips I²S (стандарт, правильний для CS4344):

LRCLK  ─────────┐                        ┌────────
                 └────────────────────────┘
SCLK   ──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──
          └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘  └─┘
SDIN   ──────[MSB][B6 ][B5 ][B4 ][B3 ][B2 ][ B1]
              ↑
              MSB через 1 такт після зміни LRCLK (gap)
```

CS4344 очікує Philips формат. Якщо надіслати Left-Justified — він отримує всі біти зсунутими на одну позицію. Результат: шум замість синусоїди.

**Рішення:** завантажувати дані в shift register на такт **після** зміни LRCLK.

`sclk_fall` — детектор спадного фронту SCLK:

```verilog
reg sclk_d = 0;
always @(posedge clk) sclk_d <= sclk_r;          // затримка на 1 такт
wire sclk_fall = sclk_d & ~sclk_r;               // було 1, стало 0 → falling edge

always @(posedge clk)
    if (sclk_fall) begin
        if      (bit_cnt == 6'd0)  shift_reg <= {left_out,  16'h0000};
        else if (bit_cnt == 6'd32) shift_reg <= {right_out, 16'h0000};
        else                       shift_reg <= {shift_reg[30:0], 1'b0};
    end
```

Один рядок різниці — `6'd0` замість `6'd63`. CS4344 одразу почав отримувати правильні дані.

## Результат

Чистий тон 440 Hz у навушниках. CS4344 модуль має вбудований підсилювач — навушники підключаються напряму.
