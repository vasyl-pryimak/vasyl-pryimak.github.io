## Нюанси PCM1808

Модуль HW-935 з PCM1808 на борту. Всі потрібні піни виведені, є вбудований LDO регулятор на 3.3V:

</a>

**Подвійне живлення:** аналогова частина — 5V, цифрова (I²S логіка) — 3.3V. Пропустити цей факт — чіп або не запустить, або дасть шум.

**Назви:** TI в документації називає тактовий вхід **BCK** (Bit Clock). В I²S це те саме що **SCLK**. Просто знати треба.

---

## I²S приймач у Verilog

PCM1808 надсилає аудіо побітово по лінії DOUT — один біт на кожен такт SCLK. Завдання FPGA: зібрати ці біти в число.

**Крок 1 — збираємо біти в зсувний регістр.**

На кожному спадному фронті SCLK біт з DOUT "заштовхується" з правого краю, а решта зсувається вліво:

```
До:    rx_buf = [b23 b22 b21 ... b1  b0 ]
Після: rx_buf = [b22 b21 b20 ... b0  dout]
```

```verilog
always @(posedge clk)
    if (sclk_rise)
        rx_buf <= {rx_buf[22:0], dout};  // зсуваємо вліво, новий біт справа
```

**Крок 2 — знімаємо готовий семпл.**

PCM1808 передає 24 біти на семпл. Коли всі 24 біти зібрані — захоплюємо верхні 16 біт:

```verilog
always @(posedge clk)
    if (sclk_fall)
        if (bit_cnt == 6'd24) left_cap <= rx_buf[23:8];  // беремо старші 16 біт
```

Чому `bit_cnt == 24`, а не `23`? Через той самий **1-bit gap** стандарту Philips I²S — MSB приходить на такт пізніше після зміни LRCLK, тому повний семпл готовий на такт 24, а не 23.

---

## Результат

Телефон → PCM1808 → FPGA → CS4344 → навушники.

Цифровий pass-through живого аудіо. Затримка — менше 1 мс.
